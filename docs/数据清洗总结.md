2.vllm版本问题

+ vllm启动qwen3出错

<u>AttributeError: Qwen2Tokenizer has no attribute all_special_tokens_extended</u>
+ 搜帖子和官方文档发现（https://huggingface.co/Qwen/Qwen3-8B和https://blog.csdn.net/weixin_64651338/article/details/158502270）：
+ <font style="color:rgb(15, 17, 21);background-color:#FBDE28;">Transformers v5.2.0 移除了 </font>`<font style="color:rgb(15, 17, 21);background-color:#FBDE28;">all_special_tokens_extended</font>`<font style="color:rgb(15, 17, 21);background-color:#FBDE28;"> 属性，而 vLLM 0.8.5 依赖该属性。Qwen3 复用了 </font>`<font style="color:rgb(15, 17, 21);background-color:#FBDE28;">Qwen2Tokenizer</font>`<font style="color:rgb(15, 17, 21);background-color:#FBDE28;"> 类名，导致冲突。</font>

- 最好的方法，降低rtansformers到4.57.3
3.评估，关于如何判断8B模型回答是否正确，

之前没有对齐的时候，根据不同的category采取了不同策略：

+ 第一种是简单事实类问题，答案很短，我用关键词匹配来判断；
+ 第二种是时间相关的问题，我先统一时间格式，再做匹配；
+ 第三种是判断性问题，答案短的用关键词匹配，长的则用语义匹配（ sentence-transformers ）；
+ 第四类是复杂问题，统一用语义匹配来评估。

<u>对齐后发现直接用14B模型评估就好了</u>

5.怎么去重呢

我的初步想法是，找到相同的QA，然后将QA相同的分为一组，给大模型评估，选择好的样例，但是发现没有完全重复的

+ 陈述事实，清晰简洁，没有干扰因素，
+ 记忆列表，从两个视角提供同一条核心信息 
+ 极其冗长和混乱，包含了大量关于其他活动（如露营、音乐会、骄傲游行、诗歌朗诵等）和大量对话引用的记忆。 
+ 信息不完整，对评估无意义
